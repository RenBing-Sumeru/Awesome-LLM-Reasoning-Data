# v0.1.0 Release Notes

## Project Goal

v0.1.0 prepares **Awesome LLM Reasoning Data** as a link-verified, searchable, learning-first atlas for post-training reasoning data. The release emphasizes official links, field navigation, card-based reading, and reproducible QA reports.

## What Changed

- Promoted the core atlas to 136 verified entries with official primary paper/arXiv/venue/DOI links.
- Added curation levels from `L0_seeded` through `L5_audit_ready`.
- Rebuilt `papers/` as a category-first paper atlas with read-first tables, full lists, audit checklists, related cards, and open gaps.
- Expanded the card library to more than 50 cards, rewrote the Beginner 20 Starter Pack cards as paper-specific audit cards, and added validation to reject placeholder language in every `L5_audit_ready` card.
- Rebuilt `docs/index.html` as a static searchable atlas powered by generated JSON plus a local fallback data bundle.
- Added export files in `exports/` and `data/citations.bib`.
- Added public coverage, needs-search, research-log, link-check, release, and self-review reports.
- Upgraded contribution guidelines, PR template, workflow checks, and validation scripts.

## Website Status

The searchable atlas is available at [docs/index.html](../docs/index.html). It loads [docs/assets/entries.json](../docs/assets/entries.json) on GitHub Pages and can fall back to [docs/assets/atlas-data.js](../docs/assets/atlas-data.js) when local browser settings block JSON loading. It supports search plus filters for year, venue, source role, verification contract, supervision granularity, training use, curation level, status, and artifact availability.

## Known Limitations

- Many entries still need official code, data, Hugging Face, project, DOI, and venue-page verification.
- Author metadata and complete BibTeX fields are incomplete for many entries.
- Soft link checking does not probe every external URL over the network.
- 63 verified entries are intentionally kept at `L1_link_verified` because their official paper link is pinned but their paper-specific summary still needs review.
- Non-core card content is intentionally conservative and may mark missing details as `unknown`; every `L5_audit_ready` card is checked against placeholder text.

## Next Milestone

v0.2.0 should continue artifact verification: code, data, project pages, Hugging Face assets, author metadata, and deeper card-specific audit notes for the highest-impact remaining entries.
