Reward supervision in post-training RL can be sparse, ambiguous, or wrong. When a noisy reward enters generalized advantage estimation, the error is not confined to one scalar: it propagates into value predictions and token-level advantages, causing unstable updates, length hacking, or a collapse in policy quality. Reward denoising and sample filtering address part of this problem but leave the value model as a passive recipient of the same unstable signal.

VRPO asks whether the value model itself can absorb and regulate noisy supervision before it drives a policy update. The paper changes the learning path from reward to advantage, rather than collecting a new reward dataset. Its focus is on the training objective and value target used to consume noisy rollout feedback in RL.
