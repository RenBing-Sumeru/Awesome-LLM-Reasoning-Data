The official paper describes the source records, applies the stated selection or verification procedure, and trains with the resulting object. The implementation should be read as a pipeline rather than a single score: collect or construct candidate records, attach the stated signal, make a keep/revise/reward decision, and train under a fixed recipe. Each stage can change the final data distribution, so reproducing only the final loss or optimizer is insufficient.

Official implementation material is linked so the record, signal, and training recipe can be inspected and reproduced. For reuse, preserve the source mixture, model version used to produce the signal, thresholds or quotas, and the unit of verification. These details decide whether a later experiment consumes the same training object or merely uses a method with the same name.
