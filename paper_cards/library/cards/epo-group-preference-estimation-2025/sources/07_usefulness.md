EPO is useful when a system can cheaply obtain multiple rollouts and validate their terminal answers, but a single chosen/rejected pair is too brittle to represent the learning opportunity. It provides a specific schema for preserving all correct and incorrect candidates and a recipe for converting that schema into a preference objective. Before reuse, teams should audit verifier error, define how duplicates and all-correct/all-wrong prompts are handled, test sample counts and truncation, and confirm that the task's terminal answer genuinely captures the behavior they want to optimize.
