TPO defines a token's visual relevance counterfactually. The same prompt and preceding response tokens are evaluated twice: once with the raw image and once with a noise-corrupted image. The difference in raw logits is the visual-anchor score. If corrupting the image changes a token's prediction substantially, the token is treated as dependent on visual evidence rather than merely on language priors.

The score is then self-calibrated into opposite reward behavior for the two sides of a preference pair. In a preferred response, visually anchored tokens receive larger reward; in a rejected response, the calibration reverses so visually anchored problematic tokens receive lower reward. Thus the method does not filter tokens once before optimization. It makes each token's contribution to the DPO-style objective dynamic and model dependent, tying the training signal to visual sensitivity.
