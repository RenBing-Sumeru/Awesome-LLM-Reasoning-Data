Use TPO when a vision-language preference dataset already has chosen and rejected answers but lacks fine-grained grounding labels. It provides a reproducible way to turn image sensitivity into a token reward and to use the same pairs more selectively. Before adoption, verify image preprocessing and corruption consistency, evaluate whether high-score tokens correspond to the visual attributes that matter in the deployment task, compare against a response-level DPO control at equal compute, and audit the licensing of the image and preference sources.
