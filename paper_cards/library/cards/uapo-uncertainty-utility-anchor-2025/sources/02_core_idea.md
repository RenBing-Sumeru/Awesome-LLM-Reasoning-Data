UAPO replaces the assumption that a pairwise comparison alone fixes the training reward with a prompt-specific reference point. The utility anchor is learned from the relationship between policy and reference-model behavior and acts as a baseline against which responses are judged. Because the anchor is conditioned on the prompt, it is not a global constant reward threshold; it can represent that the same absolute response score can have different uncertainty in different contexts.

This reference point lets the objective absorb more than strictly paired winner-loser examples. The paper describes the anchor as an uncertainty-aware signal and derives an uncertainty penalty from it. The practical consequence is that preference data can be organized in multiple forms, including unpaired data, without first forcing a synthetic pair. The contribution is therefore a decision about record eligibility and weighting, not merely a different name for an existing DPO loss.
