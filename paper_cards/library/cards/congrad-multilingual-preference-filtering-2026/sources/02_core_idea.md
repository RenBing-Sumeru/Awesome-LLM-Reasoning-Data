The method represents each language by an exponential moving average of its recent preference-training gradients. It removes mutually conflicting components from these language-level vectors with the PCGrad projection rule and sums the adjusted vectors into one consensus gradient. This makes “cross-lingual affinity” concrete: a candidate record is desirable when its own loss gradient points in a similar direction to the consensus.

The consensus is used to filter newly self-generated preference data, not to rewrite model gradients directly. For every language, CONGRAD keeps the records with the highest cosine similarity to the consensus and then applies length-penalized DPO to the retained pairs. Low-rank compression lets the procedure retain language gradient histories without storing a full dense gradient for every language, so the selection rule can be inserted into a repeated self-rewarding loop.
