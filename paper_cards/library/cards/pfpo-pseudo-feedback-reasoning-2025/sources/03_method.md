PFPO represents a task by test pairs. It samples N solutions, executes or extracts their answers, and assigns a pass rate. For outcome DPO, it retains pairs whose winner clears a quality threshold and beats the loser by a score margin. Frontier feedback uses a stronger model to obtain pseudo answers or help create test inputs; self-feedback instead uses policy-generated solutions and majority-voted outputs, permitting iterative construction.

The authors apply the procedure to mathematical reasoning and code. Their released implementation contains PFPO configuration, data processing, execution, and training code. Reported math runs include Mathstral-7B and Llama-3.1-8B; code runs use DeepSeek-Coder-7B-v1.5. They compare SFT, ordinary DPO, prefix DPO, frontier-feedback PFPO, and successive self-feedback iterations.
