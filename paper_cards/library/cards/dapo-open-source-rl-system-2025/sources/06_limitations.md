Integer-answer reward observes only the terminal value and can miss invalid intermediate reasoning or parser exploits. Dynamic sampling and large rollout workloads are sensitive to hardware, implementation, and hyperparameters, while prompt sources may overlap with benchmarks or pretraining. Reuse therefore requires a pinned data release, explicit answer parsing, compute accounting, provenance and license checks, and held-out evaluation rather than treating the public code as an unconditional reproducibility guarantee.
