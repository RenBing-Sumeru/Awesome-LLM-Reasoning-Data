Use ProDS when an SFT dataset must be tailored to a known downstream preference surface rather than merely compressed. Reserve a clean target-validation set, build and audit preference pairs, score candidate records by both directions, then compare selected SFT with random, quality-only, and target-gradient selectors under the same token budget. It is not suitable when using benchmark-derived validation data would contaminate final evaluation or when an external judge cannot be legally and reproducibly used.
