A general-purpose SFT model is trained from instruction-response data spanning domains such as instruction following, mathematics, and code. The total token budget is limited, so assigning more data to one domain necessarily reduces another. Exhaustively testing the simplex of possible mixture weights is expensive, and pre-training mixture rules may not transfer because SFT starts from a particular pretrained model and has different scaling and cross-domain-transfer behavior.

This paper asks how to select domain weights for a fixed SFT budget while minimizing held-out validation loss across the intended domains. The data object is a source-labeled instruction-response example, and the training decision is the number of tokens sampled from each domain. It does not score individual answers for correctness; it models how changing a domain's allocation affects a model after fine-tuning.
