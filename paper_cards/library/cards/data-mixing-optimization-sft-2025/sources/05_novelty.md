Earlier data-mixing work emphasized pre-training, used proxy-model weights, or treated mixture loss with formulas that do not explicitly represent SFT scaling and cross-domain transfer. This paper moves the decision to SFT and defines an effective data quantity for each target domain: its own tokens plus a fitted contribution transferred from the remaining domains. The optimizer then selects weights that minimize predicted validation loss under a finite token budget.

The new contribution is not simply fitting a curve or using SLSQP. It is the coupled loss parameterization, the perturbation protocol used to identify it, and the balanced multi-domain objective that avoids assigning all budget to a single apparently useful source. The paper's Tulu3 and Orca comparison is also part of the contribution: it shows that a good loss model is only useful when validation domains correspond to the downstream capabilities one wants to preserve.
