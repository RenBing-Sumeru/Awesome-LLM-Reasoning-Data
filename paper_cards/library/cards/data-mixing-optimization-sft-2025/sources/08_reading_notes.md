The main lesson is that data mixture is a model- and budget-dependent optimization variable, not a property of a dataset collection alone. The method's signal is validation loss at the domain level, and its usefulness rests on whether that loss is a valid proxy for the downstream contract. Read the near-grid-search result as evidence for efficient mixture search in the controlled setting, then read the Orca mismatch and IFEval exception as boundaries: domain definition, validation construction, repeated sampling, and target task alignment all determine whether the chosen weights are trustworthy.
