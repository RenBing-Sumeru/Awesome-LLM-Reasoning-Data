The novelty lies in changing the task shape of judge preference data, not simply scaling a DPO corpus. A single annotation is transformed into critique, outcome, and inverse-response records, each with a different positive/negative contract. This preserves the same judgment anchor while exposing different failure modes: incorrect reasoning, incorrect final decision, and a weak understanding of what content a decision refers to.

The response-deduction task is particularly distinctive because it asks the judge to use an evaluation as evidence about the evaluated text. Combined with the direct judgment task, it avoids treating a fluent critique as sufficient proof of a correct outcome. The result is a foundation-judge recipe whose output can be reused as a reward model and as a source of downstream preference pairs, creating a documented bridge from evaluator data to training data.
