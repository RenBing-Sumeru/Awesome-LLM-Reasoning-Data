Use this work when a post-training project has a small, high-confidence target signal but also a large general pool that should preserve broad behavior. Define the target objective and the general-data stream separately, choose a feasible-set construction that fits the memory budget, and compare projected, target-only, and ordinary mixed-data updates on held-out tasks. It is not a substitute for auditing general-data quality, and it is unsuitable when gradients, split boundaries, or the system budget cannot be controlled.
