Data-centric fine-tuning usually decides what to remove before training starts, then presents the surviving data in a random order. That practice overlooks a separate decision: the sequence of gradient updates determines how a pre-trained model moves from its original distribution toward a specialized target distribution. Random shuffling may force abrupt alternation between familiar and difficult examples even when all examples are relevant.

The paper asks whether every retained example can be used more effectively by changing when it is consumed rather than whether it is kept. It proposes an easy-to-hard first pass based on the relationship between a sample and the current pre-trained model, followed by ordinary random-order training. The goal is a smooth adaptation path that retains the full dataset rather than treating data governance only as cleaning or selection.
