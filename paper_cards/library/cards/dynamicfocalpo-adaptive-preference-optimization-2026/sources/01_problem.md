DPO uses the same preference pairs throughout a run, yet the model's relationship to those pairs changes as learning proceeds. Pairs that the implicit reward already ranks correctly can provide stable early supervision, while pairs it misranks can produce noisy or unhelpful gradients before the model has learned a basic preference boundary. A fixed focal weight assumes that this balance never changes.

DynamicFocalPO asks how an offline preference objective can control the timing of pair influence without recollecting labels or discarding hard examples forever. It treats learning stage as part of the decision about which records should dominate the update. The target is therefore the training use of an existing chosen-rejected dataset, not the construction of a replacement dataset.
