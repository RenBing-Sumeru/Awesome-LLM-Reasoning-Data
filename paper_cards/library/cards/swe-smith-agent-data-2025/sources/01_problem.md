Software-engineering agent data is often reduced to an issue–patch pair, but that discards the context that determines whether the task is real: the repository revision, dependencies, tool-visible state, and tests that judge the proposed change. Without those elements, the same text cannot reliably be replayed in the source repository, and an apparent success or failure is a weak training signal.

SWE-smith studies how to construct this executable data at scale. It turns real repositories into tasks that connect repository state, environment, tests, reference changes, and agent trajectories into a replayable work unit. The question is not whether a patch looks plausible, but whether an agent can complete the repair in a controlled environment and have the result accepted by execution.
