Use this paper to design or evaluate a coding agent that operates on an existing Python repository. Start from a released task and its pinned environment, let the agent inspect files, use tools, and submit a patch, then use the stated tests to judge success; where trajectories are available, they can provide imitation supervision for tool use. A credible experiment reports the dataset revision, whether the environment rebuilt, the test command, and the agent’s model, tools, and token/turn budget. It is not a fit for text-only code completion that cannot execute repository state, nor should a reader treat a reported target scale as a downloadable training set without checking the release.
