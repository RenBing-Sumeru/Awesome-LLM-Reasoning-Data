SWE-smith uses task tests as its terminal signal, but unit tests observe only encoded behavior: a passing patch may still violate an unstated interface constraint or maintainer intent. Its environments also depend on repository history, third-party packages, and build tooling, so an instance that ran during collection can become hard to replay after dependencies drift. Finally, source repositories and commit history can overlap with model pretraining or external coding benchmarks, mixing memorization into apparent repair ability. Before use, rebuild a pinned environment, sample-check task semantics, and audit commit dates, repository overlap, and licenses.
