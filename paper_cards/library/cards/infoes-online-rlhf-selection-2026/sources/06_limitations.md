The method has not been evaluated on 32B-or-larger models, multi-objective alignment, mathematics, or code generation. It depends on a reward model and a validation set that faithfully represent the intended objective; noise there can cause beneficial rollouts to be filtered. Per-experience gradient attribution also adds computational and storage overhead, and local per-round influence is not a guarantee of long-horizon global improvement. The paper declares future code availability, so implementation details and all upstream data licenses require verification before reuse.
