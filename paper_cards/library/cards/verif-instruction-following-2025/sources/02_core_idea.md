Its data object is an instruction paired with executable checkers, functions, and fields needed to evaluate compliance. The official VerInstruct release identifies 24K records, whereas the paper describes approximately 22K; this difference should be treated as a release-version fact to pin, not silently reconciled.

By retaining verifiers with prompts, the record states how an answer is consumed by training. It supports both deterministic checks for observable constraints and learned verification for requirements that cannot be safely reduced to a string or regular expression.
