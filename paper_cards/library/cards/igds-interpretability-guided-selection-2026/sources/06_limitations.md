IGDS assumes that a sparse feature extracted at a chosen layer and token position is a faithful handle on the desired capability; correlated features or an imperfect SAE can make a positive intervention look more causal than it is. Feature identification also needs an instruction-tuned model, task-specific validation data, and intervention computation before SFT, so it is more demanding than a one-pass loss filter. The paper does not publish a dedicated code or data release URL, which makes exact replication of SAE training, positions, and preprocessing a required check rather than an assumed capability.
