新的贡献是将回答对在记录层面转化为带 token 掩码的回答对。SePO 既不要求奖励模型标注每个动作，也不对所有回答文本不加区分地训练，而是从普通 DPO oracle 中导出可复用的 token 分数，并用它决定哪些 token 成为实际学习表面。这在不需要新的人工标注通道的情况下，改变了既有偏好数据的监督粒度。

该工作还将弱监督与弱生成区分开来。一个较小的 oracle 不必生成比目标策略更好的回答；它只需要判断哪些位置承载有用的偏好证据。这一区分支持文中报告的弱到强设置，并将该方法与直接蒸馏流程区分开来。论文的随机掩码对照使关键主张可以被证伪：增益被归因于估计的奖励排序，而不是仅仅来自训练更少 token。
