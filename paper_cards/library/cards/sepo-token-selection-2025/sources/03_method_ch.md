该过程包含三个阶段。首先，在一个中等规模的随机子集的入选回答上进行监督微调，得到参考模型。再在该子集上进行 DPO，得到 oracle 模型。对于完整目标偏好集中的每个 token，SePO 计算在给定提示和前序 token 时，oracle 概率除以参考模型概率的对数。论文认为，在其陈述的假设下，这个分数估计了被分解后的 token 级奖励。

第二步，在每条入选回答中 SePO 保留得分最高的前 k%，在每条拒绝回答中保留得分最低的前 k%。第三步，它只使用这些被掩码 token 的似然，优化一个长度归一化的对比目标。主实验使用 30% 的选择比例。作者还测试了弱到强变体：一个小型 oracle 为更强策略选择 token，或一个高质量 oracle 从弱的、分布外偏好数据中选择有用位置。
