SePO 将 DPO 本身作为奖励估计器。给定一个参考模型和一个在偏好对上经过 DPO 得到的 oracle，某个 token 在 oracle 与参考模型下的对数概率比，提供了一个与该 token 奖励贡献成比例的 token 级量。入选回答中的高比值和拒绝回答中的低比值，被视为区分该成对偏好的最有力证据。

核心动作是将这一估计转化为二元训练掩码。目标策略不再在每个 token 上接收损失，而是在最高分的入选 token 和最低分的拒绝 token 上接收对比目标。一个小型 oracle 可以为远大于它的策略模型标注训练数据，因为它识别的是有用位置，而不是直接生成目标回答。这为从粗粒度偏好转化为稀疏、可被模型消费的监督提供了一条具体路径。
