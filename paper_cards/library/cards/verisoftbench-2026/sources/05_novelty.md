What changes is It shifts formal-proof evaluation from isolated theorem statements to proof obligations that require real repository context. Deterministic verifier contract: Lean 4 type checker on repository-context proof obligations. Data/evaluation object: repository-context Lean proof obligations, dependency imports, model proof edits, and type-checker outcomes. Actual pipeline: For each obligation, the system supplies the checked-out Lean repository context; a model proposes proof code and Lean 4 type-checking supplies the terminal verdict. Quality/impact signal: Open 500-obligation software-verification benchmark from UT Austin formal-methods researchers.
