VeriSoftBench: Repository-Scale Formal Verification Benchmarks for Lean asks how repository-context Lean proof obligations, dependency imports, model proof edits, and type-checker outcomes can provide a rigorous outcome-data surface rather than a subjective model judgment. Its decision boundary is programmatically verifiable outcomes: the result is a checker verdict, not merely a persuasive natural-language answer.

Source and audit record: VeriSoftBench: Repository-Scale Formal Verification Benchmarks for Lean — https://arxiv.org/abs/2602.18307. Venue/date: arXiv; first public release: 2026-02-20. Deterministic verifier contract: Lean 4 type checker on repository-context proof obligations. Data/evaluation object: repository-context Lean proof obligations, dependency imports, model proof edits, and type-checker outcomes. Actual pipeline: For each obligation, the system supplies the checked-out Lean repository context; a model proposes proof code and Lean 4 type-checking supplies the terminal verdict. Quality/impact signal: Open 500-obligation software-verification benchmark from UT Austin formal-methods researchers. Reuse/audit caveat: Reproduce from pinned repository commits and dependency locks; audit obligation provenance, benchmark splits, and each repository license.
