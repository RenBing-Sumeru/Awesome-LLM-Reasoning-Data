Single-turn selection treats every query-answer pair as an independent record, which loses the dependencies that make a later answer useful or harmful in a conversation. MDS makes the complete dialogue the selectable record. Its global component decides whether the set covers varied user intents, while its local component decides whether the turns within a selected record provide grounded and functionally aligned supervision.

The method is not merely a generic LLM quality score applied to dialogue text. It separates user-side semantic trajectories from assistant-side structural signals, uses a hard form-consistency admission condition, and maintains bin-level quotas. This makes its contribution a concrete policy for assembling a multi-turn training consumer: conversation diversity is protected before structural reliability is used to decide which examples fill each semantic budget.
