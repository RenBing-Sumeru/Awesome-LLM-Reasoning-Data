Large multi-turn dialogue pools contain failure modes that are invisible when each turn is scored separately: later requests can drift from the original intent, assistant messages can become repetitive, and answers can ignore the form that a user asked for. Selecting a collection of individually strong turns can therefore still yield incoherent trajectories for multi-turn instruction tuning.

MDS asks which complete conversations should enter a fixed dialogue-level SFT budget. It changes the training records from an unfiltered dialogue pool into a set chosen for both coverage of user-intent trajectories and reliability within each trajectory. The intervention is not a conversational model architecture or a dialogue judge at inference time; it decides which entire conversation histories become supervision.
