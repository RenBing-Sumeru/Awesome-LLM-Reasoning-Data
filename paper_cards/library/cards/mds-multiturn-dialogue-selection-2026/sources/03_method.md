MDS encodes every user query using all-MiniLM-L6-v2, averages query embeddings across turns, runs k-means with 1,000 bins, and retains the top 50% in each bin using a greedy representativeness-minus-redundancy objective. This forms a reduced candidate pool. In the local stage, Qwen3-8B-Instruct extracts answer entities, entities in preceding queries, and entities in preceding answers; the score rewards user grounding and new informative entities. It also rates query-answer form consistency on a 0–2 scale.

The experiment uses Baize for general multi-turn training and a Banking customer-service corpus for domain training. From each corpus, MDS and all baselines select 10K dialogues, expand them into turn-level supervision, and LoRA fine-tune LLaMA3-8B-Instruct and Qwen3-8B-Instruct for three epochs. The form threshold is 1.0, per-bin quotas follow original bin size, and comparisons cover single-turn selectors, dialogue-level LLM scorers, heuristics, random, full data, and the base model. The official project releases code and resources.
