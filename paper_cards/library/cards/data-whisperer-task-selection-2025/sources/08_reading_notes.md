Read Sections 3 and 4 jointly. Section 3 defines what a score means: it is a repeated, attention-weighted ICL contribution rather than a direct estimate of label quality. Section 4 then supplies the default demonstration/query counts, task metrics, LoRA setup, and selection-to-tuning calculation needed to interpret the claimed speedups. The ablation without attention is important because the method's score-sharing assumption can otherwise confound a productive context with the contribution of every record in it. The public repository should be used to verify implementation defaults and data preprocessing.
