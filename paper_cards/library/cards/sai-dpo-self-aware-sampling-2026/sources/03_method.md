At each iteration, the current model labels the pool through repeated responses and a correctness check, producing a pass rate for each question. A sentence-transformer embedding and clustering stage assign knowledge semantics, while the error dataset reveals clusters where the learner is weak. Self-aware difficulty combines the pass rate with path complexity indicators: number of reasoning steps and answer length resolve cases with comparable pass rates. The paper uses eight rollouts for exploration and filters out all-correct and all-wrong examples.

The main offline-RL study starts from 400K Numina-Math prompts. It samples 20K prompts per iteration for eight iterations, generates preference pairs, and retains the top 70% of remaining examples after the extreme filter; the reported Qwen2.5-7B-Math-Base run consumes 48K training samples in total. It compares with iterative DPO using random samples, PPO, hard mining, curriculum, clustering-only selection, and external difficulty. It also applies the selected data to rejection-sampling fine-tuning, separating the data policy from one particular optimizer.
