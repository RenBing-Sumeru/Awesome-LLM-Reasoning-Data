SAI-DPO treats high-quality training data as learner-relative and time-varying. It maps prompts into semantic knowledge clusters, finds clusters overrepresented among the model's mistakes, and measures each prompt's self-aware difficulty from the model's pass rate together with reasoning-path step count and length. The resulting sampler is meant to focus on weaknesses without equating every hard question with a useful one.

For a selected prompt, multiple model rollouts provide correct and incorrect solutions. One correct solution becomes the preferred answer and one incorrect solution becomes the rejected answer for an offline DPO triple. The selection first removes prompts the current model always solves or never solves, then keeps the more challenging portion of the remaining distribution. After DPO changes the model, the same measurements are recalibrated, so the definition of a useful record changes with the learner.
