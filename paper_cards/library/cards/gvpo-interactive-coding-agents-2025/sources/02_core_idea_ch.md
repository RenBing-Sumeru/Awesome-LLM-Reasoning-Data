一句话贡献：GVPO 把 group policy optimization 与交互式代码环境中的 outcome/process 可验证信号结合起来。 反馈契约明确为：AppWorld goal tests plus deterministic compilation, exception, and partial-test feedback。对本分类而言，一条记录包含 AppWorld 任务状态、成组 agent rollout、代码或工具动作、编译与异常信号、部分测试和最终目标测试分数；最接近的比较基线是：只看终局的 GRPO 类奖励无法说明交互轨迹在哪一步失败，也没有确定性的中间反馈。

来源与审计记录：Group Verification-based Policy Optimization for Interactive Coding Agents — https://openreview.net/forum?id=RY47Tq0VsV。会议/日期：ICLR 2026；首次公开日期：2025-09-19。确定性 verifier 契约：AppWorld goal tests plus deterministic compilation, exception, and partial-test feedback。数据对象/评测面：AppWorld 任务状态、成组 agent rollout、代码或工具动作、编译与异常信号、部分测试和最终目标测试分数。实际流程：每个 AppWorld 任务采样一组轨迹，评测最终 goal test、编译、异常和部分测试反馈，再从这些已验证信号优化策略。 质量/影响力信号：ICLR 2026 conference paper; directly combines outcome- and process-verifiable reward signals。复用/审计注意：要核对 AppWorld 版本、任务 seed、scaffold 权限、部分测试权重，以及过程反馈是否泄漏解法。
