Group Verification-based Policy Optimization for Interactive Coding Agents asks how AppWorld task states, group agent rollouts, code/tool actions, compiler and exception signals, partial tests, and final goal-test scores can provide a rigorous outcome-data surface rather than a subjective model judgment. Its decision boundary is programmatically verifiable outcomes: the result is a checker verdict, not merely a persuasive natural-language answer.

Source and audit record: Group Verification-based Policy Optimization for Interactive Coding Agents — https://openreview.net/forum?id=RY47Tq0VsV. Venue/date: ICLR 2026; first public release: 2025-09-19. Deterministic verifier contract: AppWorld goal tests plus deterministic compilation, exception, and partial-test feedback. Data/evaluation object: AppWorld task states, group agent rollouts, code/tool actions, compiler and exception signals, partial tests, and final goal-test scores. Actual pipeline: For each AppWorld task, the method samples a group of trajectories, evaluates final goal tests plus compilation, exception, and partial-test feedback, then optimizes from the verified signals. Quality/impact signal: ICLR 2026 conference paper; directly combines outcome- and process-verifiable reward signals. Reuse/audit caveat: Check the AppWorld release, task seeds, scaffold permissions, partial-test weights, and whether process feedback leaks task solutions.
