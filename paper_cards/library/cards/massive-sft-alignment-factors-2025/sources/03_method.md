The experiment begins with ten existing SFT datasets covering code, mathematics, and general instruction behavior. For each selected model family, the authors form controlled training runs by varying dataset identity and size together with adaptation method, layer choice, epoch count, and learning-rate strategy. Each run therefore has a recoverable configuration rather than an informal recipe label.

They train more than one thousand variants and evaluate them on downstream tasks with OpenCompass. The official repository then consolidates the run metadata and scores into a dataframe: it records, among other fields, the model family, SFT data name, LoRA flag, epoch, data size, and schedule. The evaluator is the benchmark suite; the output is comparative evidence for choosing the next SFT run.
