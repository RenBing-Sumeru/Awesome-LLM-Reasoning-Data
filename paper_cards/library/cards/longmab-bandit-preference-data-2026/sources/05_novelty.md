The new object is not a generic reward model or a context-retrieval module. LongMab stores a causal construction path from chunk selection through generated answer to programmatic reward, and uses that path to decide which pair DPO will consume. Its bandit state turns repeated answer rollouts into evidence about the utility of different context segments for this particular question.

This matters because earlier long-context preference pipelines can use the whole context, fixed relevant chunks, or random chunk combinations. LongMab instead makes the sample-generation policy responsive to prior answer quality while preserving exploration. The paper's contribution is thus a data-generation policy for preference records: it decides which evidence conditions produce positive and negative answers before training, rather than only changing a DPO loss after the records exist.
