The method splits a long context into chunks and initializes each chunk's expected reward with a probe-based signal. At each of 30 rollouts, an upper-confidence-bound rule selects four 1,500-token chunks. The base model answers the question using only those chunks. The reward is the average of substring exact match on the response and F1 on the extracted answer, compared with the ground-truth answer.

For every selected chunk, LongMab updates its selection count and incremental expected reward, so later rollouts can both revisit promising evidence and try insufficiently explored chunks. The highest- and lowest-reward answers among the rollout candidates form a DPO pair paired with the original long context and question. The reported data are built from MuSiQue training questions whose contexts are augmented with random Wikipedia documents to 8K–16K tokens.
