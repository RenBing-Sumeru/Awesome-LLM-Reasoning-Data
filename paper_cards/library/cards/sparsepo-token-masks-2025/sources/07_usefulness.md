This paper is useful when a team has fixed pairwise data but wants to control where reward pressure and reference regularization act. It offers a principled alternative to treating token selection as a single static filter: keep all records, but let the objective learn which positions matter for each term. Before using it, evaluate both common and independent masks on the actual preference property, inspect sparsity by response length and token type, and verify that the selected data and hyperparameters do not shift the objective toward an unintended proxy.
