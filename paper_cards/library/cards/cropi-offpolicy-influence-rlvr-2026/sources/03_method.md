The method first stores multiple offline trajectories for each prompt under the initial policy. For the current checkpoint, it estimates an importance-weighted, unclipped GRPO-like gradient from those trajectories. Before comparing gradients, it randomly drops most dimensions and applies sparse random projection; gradient features are normalized, so practical influence is a cosine similarity rather than a length-sensitive norm. Scores from multiple selection-validation sets are combined with reciprocal-rank fusion.

CROPI divides training into phases. In each phase it scores all 47K training prompts, selects the top alpha share—10 percent in the main experiments—and performs GRPO only on that refreshed subset for a fixed number of steps. The validation data consists of up to 20 percent of designated test sets, capped at 100 examples per set; those selection examples are excluded from the reported test results. The training return is deterministic binary answer correctness with group-normalized advantages.
