1. Source and segment documents: books, arXiv papers, and code documents between 8K and 64K tokens are split into 64 chunks and clustered with BGE-M3 embeddings. 2. Generate and filter QA: Qwen3-235B-A22B proposes a question, answer, and required chunks for each cluster, judges clarity, fidelity, and evidence necessity, and two-stage rejection sampling retains one high-scoring record per document.

3. Make grounding explicit: a rollout emits selected chunk identifiers before its answer. 4. Score and optimize: the selected set receives a precision-recall F-score term, which is blended with answer reward and optimized with GRPO. The reported run trains for one epoch on 46K records, uses eight rollouts per prompt, and removes questions that the base model already answers easily; code, data, and checkpoints are released, but a full source-overlap audit is not reported.
