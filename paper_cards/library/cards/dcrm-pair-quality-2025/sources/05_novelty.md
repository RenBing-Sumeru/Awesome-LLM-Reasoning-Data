Earlier pair-selection practices commonly maximize a reward gap, reuse a fixed chosen-rejected corpus, or rely on the response source as a quality proxy. DCRM changes the selection object from an absolute preference margin to a margin-density measure. It explicitly treats lexical and model-probability distance as possible carriers of nuisance variation, so a large score is insufficient unless the score is concentrated in a relatively small total difference.

The contribution is not a new preference-optimization loss or a replacement for reward modeling. Its reusable change is the construction rule applied before the loss: from a pool of candidates, pick a pair whose judged difference is relatively clean for the current reference policy. Before reuse, a practitioner should test reward-model calibration, the cost of scoring quadratically many pairs, and whether the selected pairs inadvertently narrow response diversity or overfit the same judge.
