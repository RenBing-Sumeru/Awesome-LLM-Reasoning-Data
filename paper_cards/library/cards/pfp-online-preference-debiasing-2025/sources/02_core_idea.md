PFP first asks an LLM to infer five categories of preference features from each offline chosen-rejected example: style, tone, harmlessness, background knowledge, and informativeness. It trains classifiers that map a new instruction to feature distributions, then adjusts these assignments so the new online batch follows the seed distribution rather than the current policy's increasingly biased distribution.

The assigned features are turned into conditioning prefixes. Two prefixes generate two responses; the resulting pair is judged and added to the next preference-training batch. Sampling temperature for these prefixes decreases over iterations to make pairs progressively harder. The central idea is that feature preservation is enforced before labels are consumed, so online preference learning sees a controlled mixture of reasons for preference rather than only whichever reason has become easiest to reward.
