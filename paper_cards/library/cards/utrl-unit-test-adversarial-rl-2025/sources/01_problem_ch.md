Learning to Generate Unit Test via Adversarial Reinforcement Learning 关注如何用“程序规格、候选实现、生成的单元测试、执行轨迹、pass/fail 标签和区分性奖励”构成严格的 outcome data，而不是让模型主观评价另一个模型。它的收录边界是程序可验证结果：结论来自 checker 判定，不是看似可信的自然语言回答。

来源与审计记录：Learning to Generate Unit Test via Adversarial Reinforcement Learning — https://openreview.net/forum?id=VqjNYF9nbP。会议/日期：ICLR 2026；首次公开日期：2025-08-28。确定性 verifier 契约：Executable unit-test pass/fail and discrimination rewards。数据对象/评测面：程序规格、候选实现、生成的单元测试、执行轨迹、pass/fail 标签和区分性奖励。实际流程：策略生成单元测试，在沙箱中运行于候选程序，获得 pass/fail 和区分性结果，再用对抗奖励更新测试生成器。 质量/影响力信号：ICLR 2026 conference paper from KAIST and Microsoft Research; generated tests outperform GPT-4.1/GPT-4o in reported evaluation。复用/审计注意：要审计沙箱隔离、隐藏参考实现、奖励平衡、flaky test 过滤，以及测试是否过拟合已采样的候选程序。
