Read Sections 3 and 5 together. Section 3 defines the invariant that makes the intervention interpretable: incoming and removed records are label-balanced and the final set remains fixed at 10K. Section 5 then separates standard-OOD and challenge-OOD outcomes, which prevents the headline of improved robustness from hiding a transfer trade-off. The repository is useful for the exact data construction and API settings, but reproduce with pinned provider versions and independently audit generated examples, since the paper's core conclusion rests on the composition of a constrained training set.
