This is useful rather than essential for general-purpose curation, because it is narrowly about NLI robustness. It is highly relevant when a closed-model API fixes the learning algorithm and a team must decide what to replace within a finite paid budget. The paper supplies a disciplined replacement protocol and warns that synthetic complexity and target OOD regime must be chosen together; use it as a design reference, then validate the same trade-off on the target application rather than assuming uncertainty sampling transfers unchanged.
