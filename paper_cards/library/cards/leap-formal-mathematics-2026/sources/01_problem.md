LEAP asks how a general-purpose LLM can produce long formal mathematics proofs whose correctness is mechanically checked rather than merely persuasive in natural language. Its primary source is https://arxiv.org/abs/2606.03303, first released on 2026-06-02; the official project page is https://imobench.github.io/.

The decision boundary is Programmatically Verifiable Outcome Data: each outcome is a Lean proof artifact with a compiler/kernel verdict. It is not a natural-language math benchmark or a subjective proof-quality judgment. The evaluation object is an IMO-style formal theorem statement together with the agent's blueprint, proof edits, compiler feedback, and final proof status. This is L4-ready because the official paper, project page, and Google DeepMind code release identify both the tasks and the executable terminal contract.
