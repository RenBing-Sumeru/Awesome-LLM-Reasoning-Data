LEAP's contribution is to turn formalization into a verifier-guided agent loop instead of a one-shot proof-generation prompt. It first attempts direct formalization, then builds informal blueprints and formal proof sketches, decomposes goals into an AND-OR graph, and revises candidate Lean code using compiler feedback.

The data surface is a sequence of proof states and executable outcomes. Lean is the feedback contract: accepted subgoals can be reused in the proof graph, while rejected candidates expose machine-generated diagnostics. The closest comparisons are one-shot Lean generation, specialized theorem provers, MiniF2F/Putnam-style formal benchmarks, and other agentic provers. The direction label is blueprint-guided formal proof search with deterministic proof checking.
